## 1. Claude Code — какие Skills и MCP подключить

> **Правило:** Skills = учат Claude *как делать*. MCP-серверы = дают Claude *доступ к данным/системам*. Нужны оба типа.

### 🔌 Обязательные MCP-серверы (стартовый набор)

| MCP-сервер | Зачем | Приоритет |
|---|---|---|
| **GitHub MCP** | Читать/писать в репозитории, делать PR, просматривать историю | 🔴 Обязательно |
| **Filesystem MCP** | Работать с локальными файлами и папками | 🔴 Обязательно |
| **Obsidian MCP** | Читать и создавать заметки прямо из Claude Code | 🟠 Высокий |
| **Context7** | Свежая документация библиотек (не из обучения, а live) | 🟠 Высокий |
| **HubSpot MCP** | Доступ к CRM-данным через естественный язык | 🟡 По задаче |
| **Salesforce MCP** | Запросы SOQL, обновление сделок, аудит воронки | 🟡 По задаче |

### 🛠️ Топ Skills для подключения

| Skill | Что умеет | Установок |
|---|---|---|
| **feature-dev** | Превращает описание фичи в рабочий код | 89 000+ |
| **frontend-design** | Уникальные UI/UX решения: нестандартные шрифты, асимметрия, глубина | Топ-5 |
| **UI/UX Pro Max** | Автоматически генерирует полную дизайн-систему для проекта | 71 000 ⭐ |
| **devops-toolkit** | AWS, GCP, Datadog, Sentry — управление инфраструктурой одними командами | SRE-стандарт |
| **data-analyst** | Postgres, Snowflake, BigQuery + SQL навыки прямо в чате | Аналитика |

### ⚡ Как установить MCP-сервер (пример — Obsidian)

```bash
# 1. Установить npm-пакет сервера
npm install -g obsidian-mcp-server

# 2. Добавить в конфиг Claude Code (~/.claude/mcp_servers.json)
{
  "obsidian": {
    "command": "obsidian-mcp-server",
    "args": ["--vault", "/путь/к/твоему/vault"]
  }
}

# 3. Перезапустить Claude Code и проверить
/mcp status
```

> [!tip] Старт без перегруза
> Начни с **3 MCP-серверов**: GitHub + Filesystem + один тематический (Obsidian или HubSpot). Добавляй новые постепенно.

## 2. Совместная работа Antigravity + Claude Code

> **Статус 2026:** Antigravity 2.0 работает на Gemini 3.5 Flash (анонс Google I/O 2026). Официально поддерживает стандарт Skills от Claude Code — Skills переносятся между платформами.

### Кто за что отвечает в связке

```
ANTIGRAVITY (Большая картина)          CLAUDE CODE (Исполнение)
─────────────────────────────          ─────────────────────────
• Планирует архитектуру задачи    →    • Пишет файлы на диске
• Оркестрирует несколько агентов  →    • Делает git commit / push
• Мониторит результат в облаке    →    • Запускает тесты локально
• Работает в фоне пока ты занят   →    • Использует свой GitHub-токен
```

### Практический сценарий запуска

1. **В Obsidian** → написать ТЗ задачи в заметке
2. **В Antigravity** → скормить заметку как задачу агенту
3. **Antigravity** → строит план, передаёт в Claude Code
4. **Claude Code** → пишет код, делает коммит, пушит на GitHub
5. **Antigravity** → проверяет результат в репо, пишет отчёт в Obsidian

### Настройка гибридного воркфлоу

```bash
# Antigravity получает доступ к GitHub (токен ghp_...)
# Claude Code авторизуется отдельно:
gh auth login   # или
echo "ghp_ВАШ_ТОКЕН" | gh auth login --with-token

# Проверить что оба работают:
gh auth status
```

> [!warning] Важно
> Antigravity и Claude Code не наследуют права друг от друга. Каждому нужен собственный доступ к GitHub. Можно использовать **один токен** в обеих системах.

## 3. Улучшенная графика и UI сайтов

> **Новинка апрель 2026:** Anthropic выпустил **Claude Design** — официальный продукт для создания визуального контента.

### Инструменты по задачам

| Задача                 | Инструмент         | Особенность                                                   |
| ---------------------- | ------------------ | ------------------------------------------------------------- |
| Дизайн-система из кода | **Claude Design**  | Читает кодовую базу и создаёт систему на твоих цветах/шрифтах |
| UI-прототипы с React   | **Magic Patterns** | Handoff в Figma, дизайн-токены                                |
| Полный сайт с backend  | **Lovable**        | Код + база данных + авторизация сразу                         |
| Работа в интерфейсе    | **Nimbalyst**      | Визуальный GUI для Claude Code, kanban-доски, diff-просмотр   |

### Skill для дизайна в Claude Code

```bash
# Установить UI/UX Pro Max skill
# Добавить в ~/.claude/skills/uiux-pro.md или через /skills install

# Использование в проекте:
# "Создай дизайн-систему для сайта QSNera, 
#  используй тёмную палитру charcoal + gold accent"
```

### Что умеет frontend-design Skill

- Нестандартные пары шрифтов (не просто Roboto + Helvetica)
- Асимметричные сетки и layout
- Многослойная визуальная глубина (тени, градиенты, blur)
- Микро-анимации с правильными easing-кривыми
- Адаптивность с mobile-first подходом

## 4. Автоматизация с Obsidian

> **Статус 2026:** Экосистема MCP взорвалась до 5 500+ серверов. Claude Code 2.1.0 — +116% к автономному связыванию инструментов.

### 3 способа подключить Obsidian к Claude

| Способ | Сложность | Что умеет |
|---|---|---|
| **Obsidian MCP Server** (npm) | Средняя | Читать/писать заметки, поиск по vault, теги |
| **Obsidian Claude Code Plugin** | Низкая | Встроенный MCP-сервер прямо в Obsidian, всё в одном плагине |
| **Local REST API** | Высокая | Полный доступ к структуре vault, двусторонняя синхронизация |

### Установка Obsidian Claude Code Plugin (рекомендуется)

```
1. Obsidian → Настройки → Сторонние плагины → Обзор
2. Найти: "Claude Code MCP" (автор: iansinnott)
3. Установить и включить
4. В Claude Code: /mcp list → должен появиться "obsidian"
```

### Практические команды после подключения

```
# Сохранить итоги сессии в Obsidian
"Сохрани резюме этой сессии в мой Obsidian vault 
 в папку Отчеты/ с тегами и датой"

# Прочитать ТЗ из заметки
"Прочитай заметку 'Задача по сайту' в моём Obsidian 
 и начни реализацию"

# Создать еженедельный отчёт
"Собери все изменения за эту неделю из git log 
 и создай заметку в Obsidian/Отчеты/"
```

> [!tip] Killer-фича
> После настройки можно написать в Claude: *«Сохрани этот разговор в Obsidian»* — и получить готовую структурированную заметку с frontmatter автоматически.

## 5. ИИ-агенты для маркетинга

> **Сдвиг 2026:** Маркетинг перешёл от Chat AI (советы) к Action AI (исполнение). Claude Code автоматизирует задачи, которые раньше требовали команды.

### Что можно автоматизировать прямо сейчас

| Задача                                    | Экономия времени         | Как                                |
| ----------------------------------------- | ------------------------ | ---------------------------------- |
| SEO-аудит и отчёт                         | 3–4 часа → 10 мин        | Claude Code + браузер-MCP          |
| Генерация контента (посты, треды, статьи) | 2 часа → 15 мин          | Skill контент-маркетинга           |
| Анализ конкурентов                        | 4 часа → 20 мин          | Web MCP + структурированный промпт |
| PPC-отчёт                                 | 1 час → 5 мин            | Google Ads MCP                     |
| Персонализированные письма ABM            | Индивидуально на каждого | Claude + CRM MCP                   |

**Результаты по данным 2026:**
- 75% сокращение времени на повторяющийся анализ
- 2–3× выше response rate у персонализированных ABM-писем
- 1 человек = продуктивность 300 человек при работе через терминал

### Архитектура маркетингового агента

```
[Obsidian — задача/бриф]
         ↓
[Claude Code — анализирует бриф]
         ↓
[Web MCP — исследует конкурентов, тренды]
         ↓
[Claude Code — генерирует контент/стратегию]
         ↓
[CRM MCP — загружает в HubSpot/Salesforce]
         ↓
[Obsidian — сохраняет отчёт]
```

### Топ Skills для маркетинга

```bash
# Найти на: skillsplayground.com или claudemarketplaces.com
- content-marketing   # Посты, треды, статьи из одного инпута
- seo-audit          # Полный SEO-анализ страниц
- email-abm          # Персонализированные письма по профилю лида
- ppc-analyst        # Анализ и оптимизация рекламных кампаний
- competitor-intel   # Мониторинг конкурентов
```

## 6. Интеграция с CRM-системами

### HubSpot — самый быстрый старт (10 минут)

```
1. Зайти на: knowledge.hubspot.com/integrations
2. Найти раздел: "Claude AI connector"
3. Подключить аккаунт HubSpot → Авторизовать
4. В Claude: "Покажи мои сделки, закрытые на этой неделе"
```

**Не требует:** CLI, npm, .env файлов — только браузер.

### Salesforce — через MCP (продвинутый вариант)

```bash
# Установить официальный пакет
npm install -g @salesforce/mcp

# Добавить в конфиг MCP
{
  "salesforce": {
    "command": "salesforce-mcp",
    "env": {
      "SF_USERNAME": "твой@email.com",
      "SF_PASSWORD": "пароль+токен_безопасности"
    }
  }
}
```

**Что можно делать после подключения:**
```
"Найди все сделки в стадии Proposal за последние 30 дней"
"Обнови поле Owner у всех лидов из кампании #summer2026"
"Сгенерируй отчёт по воронке за квартал в Obsidian"
"Создай задачу follow-up для 5 лидов с просроченными активностями"
```

### Сравнение вариантов CRM-интеграции

| CRM | Метод | Сложность | Возможности |
|---|---|---|---|
| HubSpot | Нативный коннектор | ⭐ Просто | Читать/писать данные, отчёты |
| HubSpot | MCP (Composio) | ⭐⭐ Средне | + Автоматизация workflow |
| Salesforce | @salesforce/mcp | ⭐⭐⭐ Сложно | Полный доступ, SOQL, bulk-операции |
| Pipedrive | REST API + MCP | ⭐⭐ Средне | Сделки, лиды, статистика |

## 7. Организация проектов и распределение тем

> **Принцип 2026:** Context Engineering > Prompt Engineering. Хорошо настроенный контекст важнее хорошего промпта.

### CLAUDE.md — постоянная память проекта

Каждый проект должен иметь файл `CLAUDE.md` в корне. Claude читает его автоматически при каждом открытии проекта.

```markdown
# CLAUDE.md — QSNera Website Project

## О проекте
Люксовый сайт для студии плитки QSNera. Стек: HTML/CSS/JS (no framework).

## Стиль кода
- CSS переменные для всех цветов (из :root)
- Семантические имена классов
- Mobile-first

## Что нельзя трогать
- assets/ — только добавлять, не удалять
- Существующие CSS-переменные в :root

## Как деплоить
git push → автоматически через GitHub Pages
```

### Структура папок для нескольких проектов

```
~/Projects/
├── qsnera-website/
│   ├── CLAUDE.md        ← контекст проекта
│   ├── .claude/
│   │   ├── commands/    ← кастомные slash-команды
│   │   └── agents/      ← специализированные агенты
│   └── index.html
│
├── marketing-automation/
│   ├── CLAUDE.md
│   └── .claude/
│
└── crm-integration/
    ├── CLAUDE.md
    └── .claude/
```

### Субагенты для распределения задач

```bash
# Создать специализированного агента .claude/agents/designer.md
---
name: designer
description: Отвечает за UI/UX и стилистику, знает дизайн-систему QSNera
tools: [Edit, Read, Write]
---
Ты эксперт по дизайну. Всегда используй цвета из CSS-переменных проекта...
```

```bash
# Создать агента для SEO
# .claude/agents/seo-analyst.md
---
name: seo-analyst  
description: Анализирует страницы на SEO, создаёт отчёты
tools: [WebSearch, Read]
---
```

### Workflow для новой задачи

```
1. /plan          → Claude строит план без реализации
2. Отредактировать план в редакторе (исправить ошибки)
3. /implement     → Claude реализует по исправленному плану
4. Тесты          → Claude запускает и фиксит ошибки
5. git commit     → Фиксация результата
```

> [!tip] Золотое правило
> Давай Claude **одну задачу за раз**. Не «сделай весь сайт», а «исправь форму обратной связи». LLM работает лучше на фокусированных задачах.
