# 🧠 Настройка второго мозга: Antigravity + Claude Code + GitHub

> **Цель этой заметки:** Быстрый справочник по архитектуре системы, командам терминала и решению типовых проблем. Не теория — только конкретика.

---

## 🗺️ Архитектура экосистемы

```
┌─────────────────────────────────────────────────────┐
│                   OBSIDIAN                          │
│           (Идеи, задачи, документация)              │
└──────────────────────┬──────────────────────────────┘
                       │ ТЗ / заметка
                       ▼
┌─────────────────────────────────────────────────────┐
│              GOOGLE ANTIGRAVITY                     │
│    (ИИ-агент: планирует и управляет задачами)       │
└──────────────────────┬──────────────────────────────┘
                       │ команды
                       ▼
┌─────────────────────────────────────────────────────┐
│               CLAUDE CODE (Терминал)                │
│   (Пишет файлы, делает коммиты, пушит на GitHub)    │
└──────────────────────┬──────────────────────────────┘
                       │ git push / git pull
                       ▼
┌─────────────────────────────────────────────────────┐
│                    GITHUB                           │
│     (Хранит весь код со всей историей изменений)    │
└─────────────────────────────────────────────────────┘
```

| Инструмент | Роль | Аналогия |
|---|---|---|
| **Obsidian** | База знаний, постановка задач | Голова — где рождаются идеи |
| **Google Antigravity** | ИИ-агент, оркестрация | ЦНС — планирует и управляет |
| **Claude Code** | Выполнение кода в терминале | Руки — пишут и запускают код |
| **GitHub** | Облачное хранилище кода | Память — архив всех изменений |

> [!tip] Ключевой принцип
> **Antigravity** и **Claude Code** — два отдельных уровня доступа к GitHub. Каждому нужен свой токен (или один и тот же токен вставить в оба). Они не наследуют права друг от друга автоматически.

## 🔑 GitHub Personal Access Token (PAT)

### Где создать

`github.com` → Аватарка → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)** → **Generate new token (classic)**

### Какие галочки ставить

| Scope | Зачем нужен |
|---|---|
| ✅ `repo` | Полный доступ к репозиториям (читать, писать, пушить) |
| ✅ `workflow` | Управление GitHub Actions (автоматические тесты при пуше) |
| ✅ `gist` | Создание быстрых сниппетов кода |

> [!warning] Важно
> Токен отображается **только один раз** сразу после создания. Скопируй и сохрани его немедленно. Токен начинается на `ghp_...`

### Один токен или два?

**Используй один и тот же токен** для Antigravity и Claude Code — проще управлять, одинаковые права, не надо хранить два ключа. Разные токены нужны только для ограничения прав (например, стороннему боту — только чтение).

### 📦 Установка инструментов

| Команда | Что делает |
|---|---|
| `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"` | Устанавливает **Homebrew** — менеджер пакетов для macOS (нужен один раз) |

### 🔐 Авторизация GitHub

| Команда | Что делает |
|---|---|
| `gh auth login` | Интерактивный вход в GitHub (через браузер или токен) |
| `echo "ВАШ_ТОКЕН" \| gh auth login --with-token` | Авторизация по токену без вопросов (вместо `ВАШ_ТОКЕН` — твой `ghp_...`) |
| `gh auth status` | Проверить, подключён ли GitHub и под каким аккаунтом |
| `gh auth token` | Показать текущий активный токен (для сравнения последних символов) |

### 📁 Работа с репозиторием

| Команда | Что делает |
|---|---|
| `git init` | Инициализировать новый Git-репозиторий в текущей папке |

| `git status` | Показать, какие файлы изменены и что готово к коммиту |
| `git add ФАЙЛ` | Добавить конкретный файл в список изменений для коммита |
| `git add .` | Добавить ВСЕ изменённые файлы (осторожно — проверь перед использованием) |

| `git push` | Отправить локальные коммиты на GitHub |
| `git pull` | Получить последние изменения с GitHub на компьютер |
| `git log --oneline -5` | Показать последние 5 коммитов (краткий вид) |
| `git diff` | Показать, что именно изменилось в файлах |

## 🚀 Пошаговая настройка с нуля

### Шаг 1 — Установить инструменты (один раз)

```bash
# Установить Homebrew (менеджер пакетов)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установить GitHub CLI
brew install gh
```

> [!note] При установке Homebrew
> - Терминал попросит пароль от Mac — вводи вслепую, символы не отображаются
> - Если появится `Press ENTER to continue` — жми Enter

---

### Шаг 2 — Авторизоваться в GitHub (выбери способ)

**Способ A — Через браузер (проще всего):**
```bash
gh auth login
```
Отвечаем на вопросы:
1. `GitHub.com` → Enter
2. `HTTPS` → Enter
3. `Y` (Authenticate Git) → Enter
4. `Login with a web browser` → Enter
5. Скопировать 8-значный код → Enter → вставить в браузере → **Authorize github**

**Способ B — Через токен (если токен уже есть):**
```bash
echo "ghp_ВАШ_ТОКЕН_ЗДЕСЬ" | gh auth login --with-token
```

---

### Шаг 3 — Проверить подключение

```bash
gh auth status
```

✅ Успех выглядит так:
```
✓ Logged in to github.com as ВАШ_НИКНЕЙМ
✓ Git operations for github.com configured to use https protocol.
  Token: gho_...
  Token scopes: 'gist', 'read:org', 'repo', 'workflow'
```

---

### Шаг 4 — Подключить тот же токен к Antigravity

Открыть интерфейс **Google Antigravity** → Настройки MCP → поле **GitHub Token** → вставить токен `ghp_...` → **Install**

> [!tip] Как убедиться что токены совпадают
> Запусти `gh auth token` в терминале → посмотри последние 4 символа → сравни с тем, что видно в поле Antigravity (там скрыто, но последние символы отображаются). Если совпадают — всё ок.

## 🔧 Решение типовых проблем

### ❌ `command not found: gh`
**Причина:** GitHub CLI не установлен.
```bash
brew install gh
```
Если `brew` тоже не найден — сначала установить Homebrew (Шаг 1).

---

### ❌ `command not found: brew`
**Причина:** Homebrew не установлен.
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

---

### ❌ `HTTP 401: Bad credentials`
**Причина:** Токен введён неверно (лишний символ или неполный).
**Решение:** Не копируй токен вручную — используй браузерный вход:
```bash
gh auth login
# выбери "Login with a web browser"
```

---

### ❌ `Permission denied (publickey)`
**Причина:** Git пытается использовать SSH, а не HTTPS.
**Решение:** При `gh auth login` выбрать **HTTPS**, а не SSH.

---

### ❌ `fatal: not a git repository`
**Причина:** Команда `git` выполнена в папке, которая не является репозиторием.
```bash
git init  # инициализировать текущую папку
# или
cd /путь/к/папке/проекта  # перейти в правильную папку
```

---

### ❌ `remote: Repository not found`
**Причина:** Неправильный URL репозитория или нет прав доступа.
```bash
git remote -v           # проверить, куда настроен push
git remote set-url origin https://github.com/ВАШ_НИКНЕЙМ/ВАШ_РЕПО.git
```

## 📖 Словарь терминов

| Термин | Простыми словами |
|---|---|
| **Репозиторий (repo)** | Папка проекта на GitHub с полной историей всех изменений |
| **Коммит (commit)** | Точка сохранения — «сейв» с описанием что изменилось |
| **Пуш (push)** | Загрузить локальные коммиты в облако GitHub |
| **Пулл (pull)** | Скачать изменения с GitHub на компьютер |
| **Ветка (branch)** | Параллельная копия кода для безопасных экспериментов |
| **Мерж (merge)** | Слить изменения из ветки в основной код |
| **Pull Request (PR)** | Запрос на добавление изменений из ветки в main |
| **MCP** | Протокол подключения ИИ-моделей к внешним сервисам (GitHub, файлы и т.д.) |
| **ИИ-агент** | Нейросеть, которая не только отвечает, но и выполняет действия |
| **PAT (Personal Access Token)** | Личный ключ доступа к GitHub вместо пароля |
