## Ключевые выводы

### Облачный Claude Code (claude.ai/code) vs Локальный (Mac)

| | Облачный (эта сессия) | Локальный (Mac) |
|---|---|---|
| Где работает | Изолированный облачный контейнер | Непосредственно на Mac |
| Доступ к файлам | Только через Git (клон репо) | Полный, напрямую |
| Запуск браузера | ❌ | ✅ |
| Скорость итерации | Медленнее (push/pull цикл) | Быстрее (всё локально) |
| Когда удобен | Асинхронные задачи, работа с телефона | Активная разработка |

### Синхронизация
- Единственный канал между облачной сессией и Mac — **GitHub**
- Схема: изменения здесь → `git push` → `git pull` на Mac → локальный Claude Code видит всё

---

## Рабочий процесс (договорились)

1. Общаемся здесь — обсуждаем, планируем, вносим изменения
2. Перед завершением: команда **"сохрани резюме"** → я пишу в этот файл
3. Команда **"пуш"** → `git commit + git push`
4. На Mac: `git pull` → локальный Claude Code читает файл по команде

---

## Режимы Claude Code

| Режим | Поведение |
|---|---|
| **Code** (обычный, текущий) | Сразу выполняет задачи — редактирует файлы, запускает команды |
| **Plan** | Сначала составляет план, ждёт одобрения, потом выполняет |

Переключение: команда `/plan` в чате.

---

## Как передать историю чата локальному Claude Code

**Вариант 1 (автоматический):** Перед пушем сказать "сохрани резюме" — я пишу ключевые моменты в этот файл.

**Вариант 2 (ручной):** Скопировать текст чата из веб-интерфейса, вставить в файл в репо, запушить. Затем локальному Claude Code: *"Прочитай Q&A/chat-export.md — это история нашего разговора."*

**Важно:** `git push` сохраняет только файлы репозитория. История чата сессии не сохраняется автоматически — только то, что явно записано в файл.
